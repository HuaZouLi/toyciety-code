# i18n 多语言说明

ToycietyCode 的前端多语言逻辑集中在 `src/i18n/index.tsx`。当前支持英文和简体中文，语言偏好由应用设置持久化。

## 当前语言

- `system`：跟随系统语言。系统语言以 `zh` 开头时使用简体中文，否则使用英文。
- `en`：英文。
- `zh-CN`：简体中文。

对应类型：

- 前端：`src/types.ts` 中的 `AppLanguagePreference`
- 后端设置模型：`src-tauri/src/types.rs` 中的 `appLanguage`
- 默认值：`system`

## 运行时入口

应用入口通过 `I18nProvider` 注入语言上下文：

- 主应用：`src/features/app/components/MainApp.tsx`
- 独立关于窗口：`src/features/about/components/AboutView.tsx`

语言选择位于：

- `Settings > Display & Sound > Language`
- 组件：`src/features/settings/components/sections/SettingsDisplaySection.tsx`

`I18nProvider` 会根据偏好解析实际 locale，并同步设置 `document.documentElement.lang`。

## 翻译 API

从 `@/i18n` 引入：

```tsx
import { useI18n } from "@/i18n";

const { t, tx } = useI18n();
```

### `t(key, values)`

用于稳定的命名 key，适合设置导航、长期维护的共享文案。

```tsx
{t("settings.display.language.label")}
```

命名 key 定义在：

- `enTranslations`
- `zhCNTranslations`

两个对象必须保持 key 完全一致，TypeScript 会校验缺失项。

### `tx(source, values)`

用于源字符串翻译，适合已有页面迁移和大量 UI 文案接入。

```tsx
{tx("Refresh {title}", { title })}
```

中文映射定义在：

- `zhCNTextTranslations`
- `zhCNAdditionalTextTranslations`

英文 locale 会直接返回 source。简体中文 locale 会优先查中文映射，找不到时回退 source。

## 插值规则

插值使用 `{name}` 形式，值类型为 string 或 number：

```tsx
tx("Version {version}", { version })
```

翻译文本必须保留相同的占位符名称：

```ts
"Version {version}": "版本 {version}"
```

## 新增页面文案流程

1. 在组件中引入 `useI18n`。
2. 普通页面文案优先使用 `tx("English source")`。
3. 设置导航和长期共享文案可使用 `t("namespace.key")`。
4. 把简体中文翻译加入 `src/i18n/index.tsx`。
5. 动态值使用 `{placeholder}`，不要手动拼接可翻译句子。
6. 技术示例、命令、路径、URL 可以保留原样，例如 `pnpm install`、`https://github.com/org/repo.git`。

推荐：

```tsx
aria-label={tx("Close {title}", { title })}
title={tx("Refresh")}
placeholder={tx("Ask Codex to do something...")}
```

避免：

```tsx
// 不利于翻译语序调整
{tx("Close")} {title}
```

## 新增语言流程

1. 扩展 `AppLanguagePreference` 和 `AppLocale` 相关类型。
2. 在 `src/i18n/index.tsx` 增加 locale 翻译表。
3. 更新 `translations` 映射。
4. 更新 `normalizeLanguagePreference` 和 `resolveLocale`。
5. 在设置页语言下拉中增加选项。
6. 更新后端设置类型默认值和反序列化兼容逻辑。
7. 添加或更新 i18n 测试。

## 覆盖检查

直接 `tx("...")` 源文案应有中文映射。可用以下脚本检查缺失项：

```bash
node <<'NODE'
const fs = require('fs');
const path = require('path');
const root = process.cwd();
const files = [];

function walk(dir) {
  for (const ent of fs.readdirSync(dir, { withFileTypes: true })) {
    if (ent.name === 'node_modules' || ent.name.startsWith('.')) continue;
    const p = path.join(dir, ent.name);
    if (ent.isDirectory()) walk(p);
    else if (/\.(tsx|ts)$/.test(ent.name) && !/\.test\.(tsx|ts)$/.test(ent.name)) files.push(p);
  }
}

walk(path.join(root, 'src'));

const used = new Set();
for (const file of files) {
  const source = fs.readFileSync(file, 'utf8');
  const re = /\btx\(\s*(["'`])((?:\\.|(?!\1)[\s\S])*?)\1/g;
  let match;
  while ((match = re.exec(source))) {
    const raw = match[2]
      .replace(/\\n/g, '\n')
      .replace(/\\"/g, '"')
      .replace(/\\'/g, "'")
      .replace(/\\`/g, '`');
    if (!raw.includes('${')) used.add(raw);
  }
}

const i18n = fs.readFileSync(path.join(root, 'src/i18n/index.tsx'), 'utf8');
const dict = new Set();
const keyRe = /(?:^|\n)\s*(?:([A-Za-z0-9_$][\w$]*)|(["'`])((?:\\.|(?!\2)[\s\S])*?)\2)\s*:/g;
let match;
while ((match = keyRe.exec(i18n))) {
  if (match[1]) dict.add(match[1]);
  else {
    dict.add(
      match[3]
        .replace(/\\n/g, '\n')
        .replace(/\\"/g, '"')
        .replace(/\\'/g, "'")
        .replace(/\\`/g, '`'),
    );
  }
}

const missing = [...used].filter((entry) => !dict.has(entry)).sort();
console.log(`missing ${missing.length}`);
for (const entry of missing) console.log(JSON.stringify(entry));
NODE
```

## 验证命令

涉及 i18n、设置模型或页面文案时至少运行：

```bash
npm run typecheck
npm run test -- src/i18n/i18n.test.ts
```

大范围页面接入后运行：

```bash
npm run test
```

如果修改了 Rust 设置类型，运行：

```bash
cd src-tauri && cargo check
```

## 维护约定

- 不翻译用户生成内容、模型名、文件路径、命令输出、Git 分支名、远程 URL。
- 翻译按钮、菜单、placeholder、title、aria-label、toast、空状态和设置说明。
- 不把一句可翻译的话拆成多个零散片段，除非中间必须插入代码元素。
- 复用已有翻译源字符串，避免同义英文重复。
- 新增设置项时同步更新 `src/types.ts`、`src-tauri/src/types.rs`、默认设置和相关测试。
